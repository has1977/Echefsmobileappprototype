# 🐛 لوحة التصحيح - دليل كامل

## 🚀 الوصول السريع

### الطريقة الأسهل: زر البق البرتقالي 🐛
ابحث عن **الزر البرتقالي** في أسفل يمين أي صفحة!
- موضع ثابت: أسفل اليمين
- مرئي دائماً (إلا في الصفحات كاملة الشاشة)
- ضغطة واحدة → لوحة التصحيح

```
http://localhost:5173/debug-dashboard
```

## 🎯 ماذا يمكنك أن تفعل؟

### 1. **عرض حالة الجلسة الحالية**
شاهد جميع بيانات جلستك الحالية:
- ✅ الفرع المختار
- ✅ نوع الطلب (داخلي/توصيل/خارجي)
- ✅ الطاولة المختارة
- ✅ المنطقة المختارة
- ✅ عدد عناصر السلة
- ✅ عنوان التوصيل
- ✅ وقت الاستلام
- ✅ العرض المطبق

### 2. **إنشاء طلبات تجريبية**
إنشاء طلب بضغطة واحدة:

**طلب داخلي (Dine-In):**
```
- يُنشئ طلب مع رقم طاولة
- الحالة: قيد الانتظار
- النوع: dine-in
- العناصر: برجر x2، بطاطس x1
```

**طلب توصيل (Delivery):**
```
- يُنشئ طلب مع عنوان توصيل
- الحالة: قيد الانتظار
- النوع: delivery
- وقت التوصيل المتوقع: 30 دقيقة
```

**طلب خارجي (Takeaway):**
```
- يُنشئ طلب مع وقت استلام
- الحالة: قيد الانتظار
- النوع: takeaway
- وقت الاستلام: 20 دقيقة
```

### 3. **إدارة حالة الطلب**
تحديث حالة أي طلب بضغطة واحدة:
- قيد الانتظار → مؤكد
- مؤكد → قيد التحضير
- قيد التحضير → جاهز
- جاهز → قيد التوصيل (للتوصيل فقط)
- أي حالة → مكتمل
- أي حالة → ملغي

### 4. **عرض جميع الطلبات**
شاهد قائمة كاملة بجميع الطلبات في قاعدة البيانات:
- رقم الطلب
- نوع الطلب (مع أيقونة)
- الحالة (مع لون)
- اسم الفرع
- المبلغ الإجمالي
- عدد العناصر

### 5. **فحص localStorage**
عرض جميع بيانات eChefs المخزنة في localStorage:
- بصيغة JSON
- مع تنسيق ملون
- سهل القراءة

### 6. **مسح البيانات**
خياران:
- **مسح الطلبات فقط**: يحذف جميع الطلبات من قاعدة البيانات
- **مسح جميع البيانات**: يحذف الطلبات + localStorage (الخيار النووي)

## 🎬 اختبار Workflow كامل

### سيناريو: اختبار مسار الطلب الكامل

1. **افتح لوحة التصحيح**
   - اضغط زر البق البرتقالي 🐛

2. **تحقق من الحالة الحالية**
   - شاهد إذا كانت هناك أي بيانات جلسة

3. **أنشئ طلب تجريبي**
   - اضغط "إنشاء طلب داخلي"
   - يظهر الطلب في "طلبات قاعدة البيانات"

4. **حدّث حالة الطلب**
   - اضغط "تأكيد" → الطلب مؤكد
   - اضغط "تحضير" → الطلب قيد التحضير
   - اضغط "جاهز" → الطلب جاهز
   - اضغط "إكمال" → الطلب مكتمل

5. **عرض الطلب**
   - اضغط زر "عرض"
   - يفتح صفحة تتبع الطلب
   - شاهد Timeline الذكي

6. **اختبر صفحة طلباتي**
   - اضغط زر "طلباتي"
   - شاهد جميع طلباتك
   - اضغط أي طلب لعرض التفاصيل

7. **اختبر "اطلب مرة أخرى"**
   - في صفحة تتبع الطلب المكتمل
   - اضغط زر "اطلب مرة أخرى"
   - تنظف الجلسة
   - انتقل لاختيار الفرع

## 🔍 لماذا لا توجد بيانات في "طلباتي"؟

### الأسباب الشائعة:

**1. لم يتم إنشاء طلبات بعد**
```
الحل: اضغط "إنشاء طلب داخلي" في لوحة التصحيح
```

**2. الطلبات في تخزين مختلف**
```
تحقق: قسم "طلبات قاعدة البيانات" يعرض الطلبات
لكن: صفحة "طلباتي" فارغة
الحل: الطلبات قد لا تكون متزامنة مع الـ context
```

**3. مشكلة تصفية المستخدم**
```
"طلباتي" قد تصفي حسب user.id
الطلبات التجريبية تستخدم 'guest_user'
ملفك الشخصي قد يكون له user.id مختلف
```

**4. قاعدة البيانات غير مهيئة**
```
الحل: حدّث الصفحة، تحقق من Console للأخطاء
```

### الإصلاح السريع:
```javascript
// في لوحة التصحيح
1. اضغط "إنشاء طلب تجريبي"
2. اذهب لصفحة "طلباتي"
3. إذا كانت فارغة، افتح console المتصفح
4. ابحث عن أخطاء باللون الأحمر
```

## 🚨 استكشاف الأخطاء

### المشكلة: الزر غير مرئي
**تحقق:**
- هل أنت في صفحة كاملة الشاشة؟ (checkout, sign-in, إلخ)
- الزر يظهر فقط في الصفحات العادية

**الحل:**
- انتقل لأي صفحة عادية (القائمة، السلة، إلخ)
- أو استخدم رابط مباشر: `/debug-dashboard`

---

### المشكلة: لا يمكن إنشاء طلبات
**افتح Console:**
```
F12 → Console → ابحث عن أخطاء
```

**أخطاء شائعة:**
- قاعدة البيانات غير مهيئة
- بيانات الفرع مفقودة
- بنية الطلب غير صالحة

**الحل:**
- حدّث الصفحة
- تحقق من `/dev-tools` لحالة قاعدة البيانات
- امسح الـ cache وحاول مجدداً

---

### المشكلة: الطلبات لا تظهر في "طلباتي"
**التشخيص:**
1. اذهب للوحة التصحيح
2. تحقق من قسم "طلبات قاعدة البيانات"
3. إذا كانت الطلبات موجودة هناك ولكن ليس في "طلباتي":
   - مشكلة تصفية المستخدم
   - مشكلة مزامنة Context

**الحل:**
```javascript
// تحقق من user ID
console.log('المستخدم الحالي:', user);

// تحقق من الطلبات
const orders = await db.orders.getAll();
console.log('طلبات قاعدة البيانات:', orders);

// تحقق من طلبات الـ context
console.log('طلبات Context:', orders); // من useApp()
```
